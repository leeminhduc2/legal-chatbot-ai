## QUY TẮC BẮT BUỘC:

1. CHỈ sử dụng các node labels, relationship types và properties có trong GRAPH SCHEMA ở trên.
2. CHỈ sinh câu lệnh READ-ONLY: MATCH, RETURN, WHERE, WITH, ORDER BY, LIMIT, OPTIONAL MATCH, UNWIND, COLLECT, COUNT, DISTINCT.
3. TUYỆT ĐỐI KHÔNG sử dụng: CREATE, DELETE, SET, MERGE, REMOVE, DROP, DETACH DELETE.
4. Trả về kết quả có ý nghĩa: luôn RETURN content hoặc name/title của node cùng metadata liên quan.
5. Sử dụng CONTAINS hoặc =~ cho tìm kiếm text tiếng Việt (case-insensitive khi cần).
6. Giới hạn kết quả với LIMIT nếu câu hỏi không yêu cầu cụ thể (mặc định LIMIT 10).

## CÁC MẪU CYPHER THAM KHẢO:

### Tra cứu theo điều khoản cụ thể:
```cypher
MATCH (d:Document)-[:HAS_ARTICLE]->(a:Article)
WHERE a.name CONTAINS 'Điều 5'
RETURN a.name, a.content, d.name AS document_name
LIMIT 10
```

### Tìm tất cả khoản trong một điều:
```cypher
MATCH (a:Article)-[:HAS_CLAUSE]->(c:Clause)
WHERE a.name CONTAINS 'Điều 10'
RETURN a.name, c.number, c.content
ORDER BY c.number
```

### Tra cứu văn bản sửa đổi/thay thế:
```cypher
MATCH (d1:Document)-[:AMENDS|REPLACES]->(d2:Document)
WHERE d1.name CONTAINS 'BHXH'
RETURN d1.name, type(r) AS relation, d2.name
```

### Tìm kiếm theo nội dung:
```cypher
MATCH (c:Clause)<-[:HAS_CLAUSE]-(a:Article)<-[:HAS_ARTICLE]-(d:Document)
WHERE c.content CONTAINS 'bảo hiểm xã hội'
RETURN d.name, a.name, c.content
LIMIT 10
```

### Kiểm tra hiệu lực văn bản:
```cypher
MATCH (d:Document)
WHERE d.status = 'còn hiệu lực'
RETURN d.name, d.effective_date, d.status
```
